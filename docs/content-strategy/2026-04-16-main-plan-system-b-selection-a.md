# The Outcome Code 主方案

## 版本锁定

本项目当前默认执行方案固定为：

- 内容系统：`方案 B - Hybrid Radar System`
- 选题版本：`方案 A - Broad Entry Batch`

这是当前的主方案，不再并行推进其他版本，除非后续数据明确要求调整。

## 为什么锁这个组合

这个组合的核心优势是：

1. `系统层` 保留你最想做的东西
   - 常规 evergreen 主内容
   - 实时热点 pattern decode
   - bridge case 案例拆解
   - 后续可自然生长出付费层

2. `选题层` 先走最稳的英文入口
   - 不从“讲传统文化”开始
   - 从英语用户最熟悉的痛点开始
   - 先验证哪类问题真的能带来点击、收听和评论

3. 它兼顾两件事
   - 不把品牌做成一个普通的 self-help 号
   - 也不把品牌做成一个英语世界很难立刻理解的玄学号

一句话说，这个组合最适合当前阶段：

`先用最稳的英文问题入口切进市场，再用你的独特框架建立品牌辨识度。`

## 品牌一句话

`The Outcome Code is a practical English-language show that uses Chinese pattern analysis to help people read pressure, timing, character, and outcomes more clearly.`

## 前台卖点

公开内容的前台必须优先卖这些词：

- timing
- pressure
- people-reading
- anxiety
- burnout
- collapse signals
- decision mistakes
- outcomes

## 后台框架

传统文化和《易经》要作为“解释系统”出现，而不是“进入门槛”出现。

也就是说：

- 可以讲 Yin / Yang
- 可以讲《易经》
- 可以讲结构变化
- 可以讲古老模式

但必须始终服务于一个现代问题：

- 为什么我现在看不懂局势？
- 为什么我总在错误时机用力？
- 为什么某种压力会把人变形？
- 为什么结果早就在前面埋好了？

## 当前内容结构

### 1. Flagship Episode

定位：
每周 1 条 `10-14 分钟` 主内容。

功能：

- 建立主品牌
- 形成长期搜索内容
- 沉淀平台算法标签
- 作为 Shorts 的母内容

### 2. Outcome Radar

定位：
每周 `2 条` `4-8 分钟` 的热点解读。

功能：

- 跟现实世界接上
- 提供“这个号不是只讲永恒大道理”的当下感
- 用热点把新用户带进你的判断框架

必须遵守：

- 不抢新闻
- 不做立场吵架
- 不做命理预测秀
- 只做 `pattern read`

### 3. Pattern Case

定位：
每周 `1 条` `6-10 分钟` 案例拆解。

功能：

- 用人物、组织、电影、商业案例帮助用户理解你的框架
- 降低陌生感
- 增强品牌人格

### 4. Shorts Radar

定位：
每周 `6-8 条` Shorts。

结构建议：

- 3 条来自本周 flagship
- 2 条来自 radar
- 1-3 条来自 pattern case

### 5. Members / Premium

当前阶段先不重投入，但预留结构。

未来最自然的付费层不是“大课”，而是：

- `Pattern Clinic`
- `Decision Notes`
- `Pressure Playbook`
- `Weekly Brief`

## 当前优先级排序

### 第一优先级

把英语市场最容易理解的主内容做出来，并开始拿反馈。

### 第二优先级

把 realtime / bridge 内容做成固定节奏，而不是偶发灵感。

### 第三优先级

让公开内容逐步长出能转入付费的深层问题。

## 公开层的内容边界

### 要大量做的

- 时机判断
- 压力下识人
- 焦虑与错位
- 节制与边界
- 守成与失速
- 局势转折信号

### 要谨慎做的

- 纯传统文化科普
- 纯卦象解释
- 起源神话
- 太偏体系介绍的内容

### 当前不要做成主打的

- 英文算命
- 命理断事
- “我提前看穿一切”的神秘人格

## 公开层的语言纪律

### 应该多用

- pattern
- signal
- pressure
- timing
- misalignment
- overreach
- restraint
- consequence
- structure
- drift

### 应该少用

- destiny
- cosmic
- vibration
- manifestation
- secret
- prophecy

## 第一阶段的成功标准

未来 30 天内，只盯这 5 个指标：

1. 有哪类标题最容易带来点击
2. 哪类开头最能撑住前 30 秒留存
3. 评论区最常出现哪些词
4. 是 timing、pressure、people-reading 还是 anxiety 更容易起反应
5. 哪种内容更适合继续做成 bridge case 或 premium

## 第一阶段默认判断

如果前 3 周数据没有明显打脸，默认继续执行这个组合，不频繁改方向。

因为当前最大的风险不是方向错，而是：

- 换方向太快
- 还没积累 enough signal 就自我怀疑
- 还没形成内容节奏就开始二次重构

## 主方案的核心判断

`The Outcome Code` 不应该先被用户理解为“一个讲《易经》的英文号”。

它应该先被理解为：

`一个帮我看清压力、时机、人性和后果的判断型内容品牌。`

等用户已经信任你的判断力之后，再逐步接受：

- 这个框架来自中国传统模式分析
- 这个解释系统不是西方平台里已经被讲烂的那一套

这就是当前主方案存在的全部意义。
